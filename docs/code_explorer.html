Architecture Walkthrough

The run loop updates lifecycle, force, and motion systems per frame, then draws circles and FPS text. Circle lifecycle uses age and lifespan to rebirth in place while maintaining population size.

Call Graph
run orchestrates update and draw layers while helper functions isolate physics and rendering behavior.
Dependency Graph
main.py depends on pygame and Python stdlib modules for math randomness typing and dataclasses.
Full Sequence
Each frame reads events, computes dt, updates world state, draws layers, and flips display.
Frame Flow Sequence
Update order is lifecycle, then force application, then motion integration and overlap stabilization.
Lifespan Sequence
Aging exceeds lifespan and triggers immediate replacement with a fresh circle at same list slot.
Data Flow circles
circles is created once, then continuously mutated by lifecycle force motion and draw phases.
Data Flow dt
dt scales aging, repel acceleration, movement integration, and jitter probability every frame.
Data Flow age
age increases until threshold then resets through replacement circle creation.
State Model
Circle state alternates active simulation and rebirth events driven by lifespan checks.
Good Patterns
✅Separated Update Systems
Lifecycle, forces, and motion are split into dedicated functions with one orchestrator.
Basics:Smaller functions are easier to follow.
Engineering Insight:Ordered phases reduce hidden side effects.
Architecture Insight:This pattern scales feature additions.

Basics:Nearby-only checks are faster.
Engineering Insight:Shared helper avoids repeated logic.
Architecture Insight:Broad phase logic is isolated.

Basics:Types document expected values.
Engineering Insight:Static tools catch mismatches early.
Architecture Insight:Data contracts stay explicit.

Potential Issues
⚠️Dense Pair Cost
When many circles occupy nearby cells, interaction work still grows quickly.
Basics:High counts can drop FPS.
Engineering Insight:Solver passes multiply pair checks.
Architecture Insight:Adaptive pass limits can help.
Hint: stop early based on remaining overlap ratio.
⚠️Any Fallback Type
gfxdraw uses Optional Any for fallback handling, which lowers type strictness.
Basics:Any hides some mistakes.
Engineering Insight:Type checker cannot validate draw methods.
Architecture Insight:A typed adapter improves long term safety.
Hint: wrap draw backend with a small typed protocol.
